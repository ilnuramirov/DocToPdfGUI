# DOC → PDF Конвертер

Утилита на PowerShell с графическим интерфейсом для пакетной конвертации документов Microsoft Word (DOC / DOCX) в PDF с использованием COM‑автоматизации Word.

**Ключевые возможности**

- Пакетная обработка всей папки в подпапку `PDF`.
- Надёжность: 4 уровня попыток открытия (ReadOnly, без обновления связей, перезагрузка Word, отключение макросов).
- Защита от зависаний: таймауты на открытие и автоматическое завершение зависшего процесса `WINWORD`.
- Подробное логирование с временными метками.
- Оптимизированные паузы и единичный вызов сборщика мусора для скорости.

## Требования

- Windows 10/11
- Microsoft Word (Desktop) установлен
- PowerShell (Windows PowerShell рекомендуемый)
- Доступ на чтение/запись в рабочую папку

## Файлы проекта

- `DocToPdfGUI.ps1` — основной скрипт (GUI + логика конвертации)
- `Build-EXE.ps1` — (опционально) упаковка в EXE
- `Запустить.bat` — удобный ярлык для запуска
- `Build-EXE.bat` — удобный ярлык для запуска упаковки в EXE

## Быстрый запуск

Откройте PowerShell с правами пользователя и выполните:

```powershell
powershell -ExecutionPolicy Bypass -File DocToPdfGUI.ps1
```

Или дважды щёлкните `Запустить.bat`.

## Как пользоваться

1. Нажмите «Обзор» и выберите папку с `.doc`/`.docx` файлами.
2. Нажмите «Конвертировать».
3. Следите за прогрессом и логом в окне приложения. Результат появится в подпапке `PDF`.

## Что делать если файл пропущен (⚠️)

- Откройте проблемный файл вручную в Word и сохраните как DOCX.
- Убедитесь, что файл не защищён паролем и не открыт в другом приложении.
- Переместите файл в путь без длинных имён или редких символов.

## Устранение зависаний

- Закройте все интерактивные окна Word перед запуском.
- Скрипт автоматически убьёт и перезапустит `WINWORD`, если обнаружит зависание.

## Оптимизация производительности

- Запускайте на локальном SSD.
- Закройте все экземпляры Word и ненужные приложения.
- Для очень больших наборов файлов разбивайте задачу на пачки.
